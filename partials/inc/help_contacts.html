Les éléments de recherche

Pour construire votre recherche, les éléments suivants sont disponibles :

Texte simple :

Du texte rentré sans syntaxe particulière lancera une recherche textuelle, vous pouvez utiliser le caractère * si vous souhaitez faire une recherche sur le début d'un mot.

Exemple :
dupont affichera les contacts qui contiennent exactement le mot dupont (les accents et la casse ne sont pas pris en compte).
dupon* trouvera aussi bien dupont que dupond

A un e-mail :

Le mot clé :email permet d'afficher les contacts qui ont un e-mail.

A une adresse postale :

Le mot clé :adresse permet d'afficher les contacts qui ont une adresse postale.

E-mail en erreur :

Le mot clé :mailerreur permet d'afficher les contacts qui ont un e-mail en erreur.

Panier :

Le mot clé :panier permet d'afficher les contacts du panier.

Type :

Le mot clé :type1 permet d'afficher les individus.

Le mot clé :type2 permet d'afficher les structures.

Contacts :

Le mot clé :contacts(#,#,...,#) permet d'afficher les contacts ayant les identifiants correspondants

Exemple :
:contacts(1,2,3) affichera les contacts n°1, n°2 et n°3.

Etablissements :

Le mot clé :etab# permet d'afficher tous les contacts d'un établissement donnée (la fiche de l'établissement et les individus associés).

Exemple :
:etab32 affichera l'établissement n°32 et tous les individus associés.

Catégories :

Le mot clé :tag# permet d'afficher les contacts appartenant à la catégorie n°#.

Exemple :
:tag158

Départements :

Le mot clé :cp# permet d'afficher les contacts ayant une adresse postale dans le département #.

Exemple :
:cp81 affichera les contacts ayant une adresse postale dans le Tarn.

Recherches complexes

Vous pouvez combiner les éléments de recherche ci-dessus pour former des recherches complexes. Pour cela, les opérateurs suivants sont disponibles :

Réunion :

L'opérateur | (ou ^) permet d'afficher les contacts qui vérifient l'un ou l'autre des éléments de recherche (ou les deux). C'est l'équivalent du ou logique.

Exemple :
:cp12|:tag2 (ou :cp12^:tag2) affichera les contacts qui ont une adresse postale dans l'aveyron mais aussi ceux qui sont dans la catégorie n°2

Intersection :

L'opérateur & permet d'afficher les contacts qui vérifient les deux éléments. C'est l'équivalent du et logique.

Exemple :
:cp12&:tag2 affichera les contacts qui ont une adresse postale dans l'aveyron et qui sont dans la catégorie n°2.

Négation :

L'opérateur ! permet d'afficher les contacts qui ne vérifient pas l'élément.

Exemple :
!:email affichera les contacts qui n'ont pas d'adresse e-mail.

Parenthèses :

Les parenthèses permettent de regrouper les éléments pour former des recherches complexes

Exemple :
:tag2&(!:adresse|:cp81) affichera les contacts de la catégorie n°2 qui soit n'ont pas d'adresse postale, soit en ont une dans le Tarn.
